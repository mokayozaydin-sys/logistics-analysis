# Logistics Performance Analytics: A Data-Driven Approach to Sea Import

## Why Did I Develop This Project?
In the world of logistics, especially in sea import processes, data can often become very scattered and difficult to track. As an Industrial Engineer, I aimed not only to "store data" but also to create a "decision support" mechanism from that data. This project is a small end-to-end "Data Pipeline" that converts raw logistics data into a structured SQL database and then derives meaningful KPIs (Key Performance Indicators) from that data.

## Technologies I Used
While developing this project, I focused on the three fundamental needs of an engineer:

* **Python:** For data generation, analytical calculations, and visualization.
* **SQLite:** To set up a lightweight, portable, and relational database architecture.
* **SQL (DQL & DDL):** To extract meaningful metrics from the data warehouse with complex queries.

## Engineering Approach and Analysis
I did not see the project as just "writing code." To optimize cost and speed in a logistics operation, I based my approach on the following mathematical logic:

**Total cost calculation for each shipment:**
$$Total\,Cost = Freight\,Cost + Customs\,Fee + Inland\,Transport$$

When measuring port-based efficiency, I looked not only at cost but also at the correlation between **Lead Time** and cost. The key outputs produced by the project:
* Which ports are operationally slower (bottleneck)?
* Where are the cost increases concentrated along the route?
